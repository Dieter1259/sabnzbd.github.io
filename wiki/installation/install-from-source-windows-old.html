--- title: Windows run from source ---

Running the source python (.py) files on a window box is not recommended unless you want to try the latest GitHub copy, or make changes yourself.

Here is a guide to getting it working [updated June 2013].

  1. Install the latest 32bit ActivePython 2.7 from ActiveState. ActivePython is a precompiled distribution of Python. Python 3.0 is not supported and the 64bit version does not come with all that we need.
  2. Install Microsoft Visual C++ 2008 Express. You can uncheck the SQL database option during insall. Later versions of Visual Studio (2010, 2012) WON'T WORK to compile yENC, you need the 2008.1
  3. Download Cheetah.2 Extract it anywhere. Open a command prompt from the folder where you extracted Cheetah and run: setup.py install
  4. Install Git on your system.3 Defaults options are OK.
  5. Run "Git Bash" from the start menu4
  6. Create a repository clone: git clone https://github.com/sabnzbd/sabnzbd.git sabnzbd Press "Inser" instead of "Ctrl+V" to paste into Git bash. You can specify the target directory (otherwise it will be a folder called "sabnzbd" at the root of your personal files), e.g. git clone https://github.com/sabnzbd/sabnzbd.git /c/myworkingdir/sabnzbd
  7. In Git bash, go to the "sabnzbd" source folder: cd sabnzbd (or cd /c/myworkingdir/sabnzbd in our example)
  8. In Git bash, select your favourite branch: git checkout --track origin/develop for the development branch, or origin/master for the release branch. Now your SABnzbd source folder contains a copy of the branch.
  9. "Compile" the SABnzbd translations. Open a command prompt from the folder "sabnzbd" and run: tools\make_mo.py
  10. If not already present, copy the directory named "win" from sabnzbd windows binaries to your "sabnzbd" source folder. This folder contains the par2, unrar and unzip executables.5
  11. Optional (for NNTP/HTTPS support): Download pyOpenSSL version 0.11 (or higher - version 0.13 has indeed been reported to work). Choose pyOpenSSL-0.13.winxp32-py2.7.msi for the 0.13 version of pyOpenSSL if you're running ActivePython version 2.7. Launch the msi and proceed through the installation wizard with the default options.
  12. Optional (for performance improvement): Download yENC Win32 module for Python (click on the "Branches" tab then "zip"). Extract it anywhere. Open a command prompt from the folder where you extracted yENC and run: setup.py install
  13. Launch SABnzbd.py

Optional (not show the DOS command line when SABnzbd is launched):

  • Create a .cmd file to easily launch SABnzbd such as: python "path\to\folder\SABnzbd.py"
  • Put a shortcut to the .cmd file in your startup folder, and set it to start as minimized.

Optional (later SABnzbd source updates):

  • To update your SABnzbd source code from the Github repository, run: git pull
Footnotes
1. Cygwin and Allessandro Duca's original yEnc module for Python could also be used instead of MS Visual C++ and the modified KrisWebDev's yEnc Win32 module for Python. This would require a longer install and tweaks. MinGW won't work to compile yENC or would require even more complex tweaking.
2. DON'T use pypm to install Cheetah. Cheetah's _namemapper.c will not compile if you use pypm. If you don't know what is pypm, forget about this footnote, you don't need to use it.
3. If you don't know Github, you should read this short explaination
4. If you are behind a proxy, follow these steps on Git bash.
5. You could also compile par2cmdline, unrar and unzip from source but that's not the purpose of this tutorial.
