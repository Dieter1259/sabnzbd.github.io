--- title: Special settings ---

There are some special settings that have no fancy Web user interface.
The explanation of each is here.

Field name Default Meaning
start_paused 0 1 will mean that SABnzbd will always start up in "paused" mode
no_penalties 0 Some Usenet servers have intermittent login (or other) problems. For these the server blocking method is not very favourable. Setting this option will limit blocks to 1 minute.
ignore_wrong_unrar 0 On Linux, SABnzbd checks whether the installed unrar or rar program is supported and issues a warning if not. Set this option if you think you know better.
create_group_folders 0 Older versions of SABnzbd had the possibility to move downloads to folders named after the actual Usenet group. This option is made more or less obsolete by the category system. If you still want the old behaviour, remove all categories and set this option to 1
queue_complete_pers 0 Set this to 1 if you want end-of-queue actions Shutdown/Standby/Hibernate to be persistent (remembered) after restarts
api_warnings 1 When the API is accessed by external tools without proper authentication, SABnzbd will show a warning. This is a security warning that you should take seriously. However, it seems that there are some security tools that try to access anything that looks like a webserver. If you cannot disable the cause of the warnings, you can choose to disable the warning.
allow_64bit_tools 1 If your 64bit Windows system has problems running the 64bit releases of par2 and unrar, you can force the use of the 32bit releases.
par2_multicore 1 If your system has trouble running the multi-core version of the par2 program, set this to 0
never_repair 0 Only perform QuickCheck and par2-verify, but never par2-repair. Use this when your low-performance system has trouble running repairs. All available par2 files will be downloaded, so you can let another system do the repair. If the download verifies OK, post-processing will be done as usual.
allow_streaming 0 Experimental option allowing streaming. You'll need a mediaplayer that can play from an incomplete set of unprocessed files. Jobs will not be post-processed, but remain in the temporary download folder and finally end up in History as "failed". From there you can either post-process the job after all or just delete everything.
ignore_unrar_dates 0 Normally unpacked files will get the date/time stamp they have in the archive. If you set this option, the files will get the date/time of unpacking.
rss_filenames 0 Normally the RSS titles are used to name jobs. If you prefer the (usually) more compact NZB file names that are generated by index sites, set this to 1. The RSS matching still occurs on titles only!
osx_menu 1 Set to 0 to remove the SABnzbd icon in the OSX menu bar
osx_speed 1 Set to 0 to prevent the display of download speed adjecent to SABnzbd's icon in the OSX menu bar
win_menu 1 Set to 0 to remove the SABnzbd icon from the Windows systray
uniconfig 1 Set this to 1 if you try out the universal config. Note that you must obtain the config.rar and extract it's contents into the SABnzbd interfaces folder.
use_pickle 0 Only set to 1 when there is a problem with one of the standard Python libs: [0.6.15+ only]
allow_incomplete_nzb 0 Sometimes you get an incomplete NZB file (partial content, incorrect syntax etc.) You can enable this option to allow SABnzbd to salvage as much as possible from the NZB. The job will enter the queue in paused mode.
random_server_ip 1 Some Usenet providers have multiple IP addresses for a single server name. For such a server, SABnzbd will pick a random IP from the list for each connection. If your server has problems with this (e.g. it also lists invalid IPs), you can set this option to 0
size_limit 0 Any download that exceeds this value will be paused at get priority "low". You can use the K/M/G notation, so e.g. 4.7G
folder_max_length 128 On Windows, the total path is limited to 255 characters. A combination of long folder names and long file names may lead to problems. Set this option to limit each element of the total path. Each element will be truncated to the specified number. 64 is a safe value. The option will also work for other operating systems.
fsys_type 0 Only for Unix/Linux systems. 0 = Determine automatically, 1 = Names are stored as 8-bit ASCII characters, 2 = Names are stored as UTF-8. Use this only when the names created by SABnzbd look distorted! It will only work for folders created by SABnzbd itself, not files and folders created by unzip and unrar.
movie_rename_limit 100M The minimum size of a file to trigger file renaming in the Sort functions
nomedia_marker Just before unpacking of files into the final folder, SABnzbd will put a special marker file (e.g. ".nomedia") in the folder. Afterwards it will be deleted. This prevents some mediaplayer software from trying to index the folder while still incomplete. You can disable the feature by clearing the field, or you can set an alternative name. Check your media software's manual.1
req_completion_rate 100.2 Minimum percentage to allow pre-check to continue downloading.
wait_ext_drive 5 When SABnzbd wants to store files on the final destination, this destination might be an external drive. Especially USB drives can take considerable time to wake up after going into standby. SABnzbd will try 5 times with intervals of one second. You can increase this value if needed. [0.7.1+ only]
no_ipv6 0 Some systems have problems dealing with the IPv6 equivalent of "localhost". If so, set this to 1. Please note that you probably won't get this far, so you might have to use the command line parameter â€”no_ipv6 once.
rss_odd_titles see text Some RSS feeds produce very awkward titles which are fine for filtering, but not as job titles. For all indexers that are listed here, the actual NZB file name will be used instead of the title. The default is a comma-separated list of URLs and the default is: nzbindex.nl/, nzbindex.com/, nzbclub.com/ [0.7.4+ only])
overwrite_files 0 When you store downloads in a single folder or in Season folders, existing files will not be overwritten, but the new files will get a modified name. If you often download improved versions of the same item, you'll end up with many alternative versions on your disk. By setting this option to 1, existing files will simply be overwritten instead. [0.7.4+ only]
show_sysload 2 When working on a Linux/Unix system you can have SABnzbd display no system information (0), Only the CPU load indicators (1) and also the memory usage (2)2 [0.7.4+ only]
prio_sort_list A list of file name extensions. Matching files will be the first to be downloaded within an NZB. Example: sfv, nfo [0.7.5+ only]
empty_postproc 0 Do post-processing and run the user script even if nothing has been downloaded. This is useful in combination with tools like SickBeard, for which running the script on an empty or failed download is a trigger to try an alternative NZB. Note that the "Status" parameter for the script will be -1. [0.7.5+ only]
ipv6_servers 1 Control the use of IPv6 Usenet server addresses. Only useful when you have an IPv6 capable ISP and connect to servers that (also) have IPv6 addresses. Meaning: 0 = don't use, 1 = use when available and reachable, 2 = force usage (when SABnzbd's detection fails) [0.7.6+ only]
wait_for_dfolder 0 Some people use external or network drives for the "temporary download folder". It can happen that such a drive isn't mounted yet when SABnzbd starts up, causing it to create a new folder at the default location. To prevent this, set the option to 1. This will make SABnzbd wait until the drive is available. Note that SABnzbd will hang until the drive is available! Also, the folder must already exist otherwise SABnzbd will hang until terminated. [0.7.8+ only]
enable_recursive 1 1 will make SABnzbd unpack archives within archives. 0 will prevent this. [0.7.13+ only]
sanitize_safe 0 1 will make sure that SABnzbd will not use characters that Windows cannot handle in folder names. Useful when running SABnzbd on a Linux-based file server. Windows clients would otherwise have problems accessing some folders. [0.7.15+ only]
flat_unpack 0 1 will make unrar and unzip ignore any folders embedded in ZIP and RAR files. [0.7.17+ only]
warn_dupl_jobs 1 1 will issue a warning when a duplicate job is detected and subsequently paused or deleted. Set to 0 if you find the warnings a nuisance [0.7.17+ only]
enable_meta 1 1 will enable meta-tags coming from indexers or from the content of the NZB. These tags contain cleaned up versions of titles, episode names, etc. If you prefer that everything is derived from the NZB name, set this to 0 [0.7.17+ only]
new_nzb_on_failure 1 1 will enable the X-Failure header. This header is sent by some indexers and will point to an alternative NZB, should the current one fail to complete successfully
history_limit 10 Some external tools query the full history repeatedly, without setting a limit to the amount of items. With a large history, this can lead to problems. Set this option to limit the history.

Very special option

Below is one option that cannot be set from the Web UI, because it would defeat its security purpose.
You can only access it by directly editing the sabnzbd.ini file, while SABnzbd is not running.

Field name Default Meaning
config_lock 0 If 1, forbids all access to the Config pages.
Footnotes
