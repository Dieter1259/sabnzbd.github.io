--- title: Configure ---

SABnzbd comes with sensible defaults to work, however there are plenty of options and tweaks that may be done to get the most out of what sab can do.
View the relevant page for more information on what an option does and more.

All configuration data is stored in the so-called INI file. Which INI file used, you can see on the Configuration main page.
The default location is:

Windows XP/2003 C:\Documents and Settings\user\Local Data\Application Data\sabnzbd\sabnzbd.ini
Windows Vista, 7, 8 or 10 C:\Users\user\AppData\Local\sabnzbd\sabnzbd.ini
Unix $HOME/.sabnzbd/sabnzbd.ini
OSX /Users/user/Library/Application Support/SABnzbd/sabnzbd.ini

Sab relies on using existing utilities to extract, repair, and process files once they have been downloaded.
Some of these utilities are provided with sab while some require the user to install, also some may only be optional depending on how you use sab.

SSL

In order to connect to your newsserver using SSL you need to install pyOpenSSL and OpenSSL (sometimes provided in 1 package). Installation depends on your platform.
Windows and OSX packages already include the required files and should work after installation of SABnzbd.
Ubuntu: sudo apt-get install python-openssl
Running from source: pip install pyopenssl

yEnc

Posts on usernet are coded in the yEnc format, decoding this during the downloading using Python is not very efficient. Therefore we implemented this part in C for a big gain in performance.
For your system you might need to compile this module separately.
